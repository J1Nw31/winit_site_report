# WINIT 站点二维码报修系统使用指南

## 一、系统用途

操作员扫描站点二维码后，网页会自动识别站点编号，例如 `LS01`。

操作员可以：

- 不填写问题描述，直接快速发送报修
- 打开“添加问题描述”开关，填写详细故障内容后发送

维保人员会通过信息平台收到站点编号和问题描述。

---

## 二、操作员报修方法

### 快速报修

1. 使用 PDA 或手机扫描对应站点二维码。
2. 确认页面中的站点编号正确，例如 `LS01`。
3. 保持“添加问题描述”开关关闭。
4. 点击红色的“发送报修”按钮。
5. 页面显示发送成功后，等待维保人员响应。

未填写问题时，维保通知会显示：

```text
问题：未填写，现场请求维保
```

### 填写详细问题

1. 扫描站点二维码。
2. 打开“添加问题描述”开关。
3. 在输入框中填写故障现象。
4. 点击“发送报修”。

例如：

```text
输送线停止，复位后仍无法启动
```

### 发送冷却

发送成功后，按钮会进入 20 秒冷却时间。

冷却结束前不能重复发送，避免维保人员收到大量重复通知。

---

## 三、站点二维码网址格式

网站地址：

```text
https://hik2.tail6f1a46.ts.net/web-site-report/
```

不同站点通过网址中的 `site` 参数区分：

```text
https://hik2.tail6f1a46.ts.net/web-site-report/?site=LS01
https://hik2.tail6f1a46.ts.net/web-site-report/?site=LS02
https://hik2.tail6f1a46.ts.net/web-site-report/?site=LS03
```

二维码必须使用完整网址，不能只包含 `LS01` 或 `=LS01`。

---

## 四、快速生成新站点二维码

项目目录：

```text
C:\Winit\web_site_report
```

### 双击生成

1. 打开 `C:\Winit\web_site_report`。
2. 双击：

```text
快速生成站点.cmd
```

3. 输入单个站点：

```text
LS02
```

或一次输入多个站点：

```text
LS02,LS03,LS04
```

4. 工具会自动生成二维码并打开 `qr` 文件夹。

生成结果示例：

```text
C:\Winit\web_site_report\qr\LS02.png
C:\Winit\web_site_report\qr\LS03.png
C:\Winit\web_site_report\qr\LS04.png
```

### PowerShell 生成

也可以在项目目录运行：

```powershell
.\Generate-Site-QR.ps1 -Site LS02,LS03,LS04
```

站点编号格式：

- 2 至 6 个英文字母
- 后面跟 2 至 4 位数字

有效示例：

```text
LS01
RS14
PACK01
```

---

## 五、打印和张贴二维码

1. 打开 `qr` 文件夹中的 PNG 图片。
2. 打印二维码，建议尺寸不小于 4 × 4 厘米。
3. 在二维码旁标明站点号，例如 `LS01`。
4. 将二维码张贴在对应站点容易扫描的位置。
5. 张贴前使用 PDA 扫描一次，确认页面显示正确站点号。

如果重新生成过二维码，请停止使用旧二维码并重新打印。

## 六、维保人员接收通知

维保人员需要在信息平台中订阅：

```text
离线工作站报障
```

通知内容示例：

```text
LS01

输送线停止，无法重新启动
```

## 七、常见问题

### 页面可以打开，但没有自动显示站点号

检查浏览器地址是否包含：

```text
?site=LS01
```

正确示例：

```text
https://hik2.tail6f1a46.ts.net/web-site-report/?site=LS01
```

如果使用的是旧二维码，请重新运行“快速生成站点.cmd”并重新打印。

### PDA 点击“发送报修”没有反应

1. 关闭原页面。
2. 重新扫描二维码。
3. 确认 PDA 已连接网络。
4. 尝试刷新页面。
5. 检查 PDA 的日期和时间是否正确。

当前网页已针对旧版 PDA 浏览器使用兼容 JavaScript。

### 页面提示发送失败

检查：

- PDA 是否能够访问互联网
- 是否能够打开 `https://hik2.tail6f1a46.ts.net/`
- 公司网络是否屏蔽信息平台
- 站点号格式是否正确

### 通知没有到达维保手机

检查：

- ntfy 应用是否允许通知
- 是否订阅了正确 Topic
- 手机是否处于省电或勿扰模式
- ntfy 应用是否被系统停止后台运行

## 九、重要注意事项

- 每个实体站点必须使用自己的二维码。
- 打印前必须确认二维码打开的站点编号正确。
- 不要将一个站点的二维码张贴到另一个站点。
- 不要随意修改 `config.js` 中的信息平台地址、频道或密码。
- 网站源码更新后，应确认线上页面已更新再重新生成二维码。
